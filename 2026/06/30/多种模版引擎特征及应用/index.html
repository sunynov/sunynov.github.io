Python模板引擎

Bottle

语法

Bottle使用SimpleTemplate引擎或简称为 stpl，支持%语法。

SimpleTemplate 支持 Python 表达式，也因此容易出现 SSTI。

  • 使用 类似 Python 的语法（表达式、方法、属性访问）。
  • 支持 变量替换：{{ var }}。
  • 支持 控制结构（以 % 开头）：例如 %if、%for、%while（常用 %if、%for）。
  • 默认不提供严格的沙箱环境 —— 即模板中通常可以执行 Python 表达式 / 调用内建对象，存在 SSTI 风险。

特性

支持斜体、全角、八进制，非常容易绕过WAF

Mako

语法

  • 使用 类似 Python 的语法（表达式、方法、属性访问）。
  • 支持 变量替换：${ var }。
  • 支持 控制结构（以 % 开头）：例如 %if、%for、%while（常用 %if、%for）。
  • 默认不提供严格的沙箱环境 —— 即模板中通常可以执行 Python 表达式 / 调用内建对象，存在 SSTI 风险。

PHP模版引擎

Twig

特性

  • 使用统一的属性/方法访问语法：使用点号 . 可以动态访问数组的键、对象的属性，或者调用对象的方法（例如 user.name 既可以代表数组的键值，也可以代表类属性或 getName() 方法），由引擎在底层自动进行判断和兼容。
  • 支持求值与变量替换：使用双大括号 {{ var }}。它会计算括号内表达式的值并将其嵌入到页面中，且默认会自动对输出内容进行 HTML 实体转义。
  • 支持标签控制结构：使用 {% %} 标志。专门用于编写逻辑控制（如条件判断 {% if %}`、循环遍历 `{% for %}`），由于不依赖缩进，它强制要求使用显式的 **闭合标签**（如 `{% endif %}、{% endfor %}）来划定代码块边界。
  • 采用严格的默认安全隔离环境：模板运行在专门的沙箱机制中，默认完全禁止直接调用原生的 PHP 危险函数（如 system()、eval()）。发生 SSTI 风险时，通常需要依赖特定版本下暴露的内置魔术对象（如 _self.env）或特定的高级过滤器（如 filter、map）作为跳板来间接执行代码。
